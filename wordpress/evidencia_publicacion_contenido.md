# Evidencia de Publicación de Contenido

## Objetivo

Verificar el correcto funcionamiento del sitio WordPress mediante la creación y publicación de contenido.

---

# Acceso al Panel Administrativo

Se ingresó al panel de administración utilizando la URL:

```
http://54.165.81.84/wp-admin/about.php```

Se inició sesión con el usuario administrador creado durante la instalación.

---

# Creación de una Entrada

Se creó una nueva entrada con la siguiente información:

**Título**

```
Implementación de WordPress en AWS
```

**Contenido**

Se describió el proceso de despliegue del sitio WordPress utilizando Amazon EC2, Amazon RDS, Amazon S3 y Application Load Balancer, validando la correcta integración de todos los servicios.

---

# Publicación

La entrada fue publicada correctamente desde el panel de administración.

Posteriormente se verificó que el contenido fuera visible desde la página principal del sitio.

---

# Validaciones Realizadas

Se comprobó que:

- El sitio responde correctamente.
- La base de datos almacena la información.
- WordPress recupera el contenido desde Amazon RDS.
- Los archivos multimedia pueden almacenarse en Amazon S3.
- El sitio permanece disponible mediante el Load Balancer.

---

# Evidencias

Se recomienda incluir las siguientes capturas:

- Pantalla de instalación completada.
- Panel de administración de WordPress.
- Entrada publicada.
- Página pública mostrando la publicación.
- URL del sitio funcionando correctamente.

---

# Resultado

La publicación de contenido fue exitosa, demostrando que toda la infraestructura implementada en AWS funciona correctamente y que WordPress puede administrar contenido de forma estable y segura.
